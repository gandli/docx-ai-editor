# 合规性检查场景示例

## 场景概述

本示例展示如何使用 DOCX AI Editor 对采购文档进行全面合规性检查，识别违反政府采购法规和内部管理制度的问题。

## 测试文档

| 文档名称 | 文档类型 | 主要合规问题 |
|---------|---------|-------------|
| `emergency-procurement.docx` | 紧急采购 | 程序简化合规性 |
| `international-supplier.docx` | 国际采购 | 跨境合规、外汇管理 |
| `multi-vendor-bid.docx` | 多供应商联合投标 | 联合体合规性 |

## 预期发现的问题

### 🔴 高风险违规

#### 1. 紧急采购程序滥用风险
**适用文档**: `emergency-procurement.docx`

- **违规描述**: 紧急采购程序使用条件不够明确，存在滥用风险
- **法规依据**: 《政府采购法》第三十条 - 紧急采购仅限于"紧急需要"
- **风险等级**: 高
- **具体表现**:
  - "预计未来72小时"的表述过于主观
  - 缺乏第三方权威机构认定
  - 未说明为何不能采用常规采购程序

**AI 建议**:
```
建议补充以下内容：
1. 气象部门或应急管理部门的正式预警文件
2. 说明常规采购程序无法满足时间要求的具体原因
3. 记录紧急采购决策的会议纪要
```

#### 2. 国际采购外汇合规风险
**适用文档**: `international-supplier.docx`

- **违规描述**: 未明确外汇支付审批流程
- **法规依据**: 《外汇管理条例》、《政府采购进口产品管理办法》
- **风险等级**: 高
- **具体表现**:
  - 缺少外汇管理局审批文件引用
  - 未说明进口产品必要性论证
  - 汇率风险承担方不明确

### 🟡 中风险违规

#### 3. 联合体投标合规问题
**适用文档**: `multi-vendor-bid.docx`

- **违规描述**: 联合体协议要求不够详细
- **法规依据**: 《招标投标法》第三十一条
- **风险等级**: 中
- **具体表现**:
  - 未要求提供联合体共同投标协议
  - 未明确联合体各方资质认定标准
  - 缺少联合体变更或退出机制

#### 4. 信息公开不充分
**适用文档**: 多个文档

- **违规描述**: 部分关键信息公开不完整
- **法规依据**: 《政府采购信息公告管理办法》
- **风险等级**: 中
- **具体表现**:
  - 评标标准权重设置不够透明
  - 未公布质疑投诉渠道
  - 合同主要条款公开不完整

### 🟢 低风险问题

#### 5. 文档格式不规范
- **问题描述**: 部分文档格式不符合政府采购文件标准格式
- **风险等级**: 低
- **建议**: 统一使用政府采购文件模板

## 合规检查清单

### 程序合规性
- [ ] 采购方式选择符合法定条件
- [ ] 审批程序完整
- [ ] 公告发布符合时限要求
- [ ] 评标委员会组成合规

### 内容合规性
- [ ] 资格条件设置合理
- [ ] 评标标准量化可执行
- [ ] 合同条款符合法律规定
- [ ] 付款方式合规

### 国际采购特殊合规
- [ ] 进口产品审批手续
- [ ] 外汇支付审批
- [ ] 海关报关要求
- [ ] 国际仲裁条款

### 紧急采购特殊合规
- [ ] 紧急情况认定文件
- [ ] 决策程序记录
- [ ] 事后审计安排
- [ ] 信息公开补救措施

## AI 审查对话示例

### 用户提问
```
这份紧急采购文件是否符合政府采购法规？
```

### AI 回复
```
经审查，该紧急采购文件存在以下合规问题：

**⚠️ 高风险问题：**
1. 紧急事由认定依据不足
   - "预计未来72小时"缺乏权威机构认定
   - 建议：附气象部门红色预警文件

2. 程序简化理由不充分
   - 未说明为何不能采用竞争性谈判等较快程序
   - 建议：补充程序选择论证

**📝 中风险问题：**
3. 信息公开不充分
   - 紧急采购也需要事后公告
   - 建议：明确公告时间和方式

**✅ 合规亮点：**
- 设置了5日交付的合理期限
- 保留了验收和付款环节

**建议操作：**
1. 补充紧急情况认定文件
2. 完善决策程序记录
3. 制定事后公告计划

是否需要我生成合规整改建议书？
```
